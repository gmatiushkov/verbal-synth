---

## 1. Тема ВКР

**Разработка интеллектуальной системы для автоматической настройки параметров синтеза
звука с применением методов машинного обучения**

- Специальность: Программная инженерия
- Кафедра: компьютерных технологий
- Тип работы: бакалаврская выпускная квалификационная работа (ВКР)
- Проект (кодовая база): VerbalSynth — вокодер-синтезатор + ML-модуль «Generate»
  (текст → параметры/пресет звука).

---

## 2. Методология работы над запиской

Работаем **итеративно, сверху вниз** — от структуры к деталям:

1. **Оглавление** — определяем структуру разделов.
2. **Наполнение в общих чертах** — что будет в каждом разделе; при необходимости вводим подпункты.
3. **Углубление** — постепенно решаем содержание каждой части, спускаясь к деталям.
4. **Вложения** — по ходу определяем, где нужны рисунки, таблицы, приложения, листинги.
5. **Финальный текст** — окончательные абзацы пишем ближе к концу.

### Слои детализации (каскад файлов)
Каждый уровень детализации — отдельный файл; верхний = источник истины для нижнего. Чтобы
изменить что-то, «отдаляемся» до нужного уровня, правим там и перегенерируем нижние уровни.

- `L1_structure.md` — **L1**: оглавление (разделы/подразделы).
- `L2_outline.md` — **L2**: наполнение в общих чертах (что/зачем + вложения).
- `L3_detailed.md` — **L3**: углубление (развёрнутые тезисы, конкретика). _(возможно разбиение по главам)_
- `L4_text/` — **L4**: финальный текст по главам.

`CONTEXT.md` — мета-документ (методология, решения, статус), вне каскада.

### Инструментарий (решено 2026-06-16)
- **Формат документа:** пишем в **Markdown** в репозитории; Word/.docx по ГОСТ собирает автор вручную в конце.
- **Схемы:** **Graphviz** (блок-схемы — архитектура, сигнальная цепь, конвейеры).
- **Графики:** **matplotlib** (ADSR, спектры, метрики). Код генерации + вывод PNG/SVG — в `thesis/figures/`.

Сопутствующие активности по мере необходимости:
- дополнительные исследования, тесты, сбор статистики/данных из кода для аналитики;
- изучение кодовой базы и обращение к истории / ходу выполнения проекта;
- автоматизация генерации схем/графиков кодом (Graphviz + matplotlib);
- коммиты по ходу прогресса.

## 3. Цели и принципы текста

**Главная задача записки — донести «что студент сделал».**

Принципы:
- **Интересно, логично, не переусложнено** — ни научно, ни музыкально.
- **Понятно неспециалистам**: большинство членов комиссии не связаны ни с музыкой,
  ни с конкретной предметной областью студента.
- **Плавный ввод читателя** в суть проделанной работы.
- **Отражение процесса проектирования** программной системы: объяснение и обоснование
  принятых по ходу решений (архитектура синтезатора, назначение блоков, схемы/диаграммы).
- **Поэтапность** проектирования как сквозная линия повествования.
