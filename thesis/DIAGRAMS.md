### Рис. 3.5 — Каскад L1→L4 интеллектуального подбора
- Тип: конвейер / поток данных (data-flow), линейная цепочка слоёв слева направо.
- Содержание: как текстовый запрос превращается в 38 числовых параметров синтезатора;
  показывает четыре слоя обработки (L1→L4) и где находится **обучаемое ядро** (L2).
- Составные части (блоки-слои):
  - вход — текстовый запрос пользователя («тёплый бас, ярче»);
  - **L1** — текстовый энкодер **e5-base** (заморожен) → числовой вектор (эмбеддинг);
  - **L2** — обучаемая «голова» (MLP) → распределение по 140 прототипам → выбор прототипа (выделить как ядро);
  - **L3** — таблица прототипов (`library.json`) → готовый набор 38 параметров выбранного прототипа;
  - **L4** — блок параметрических модификаторов (слова «ярче/короче/…») → сдвиг затронутых параметров;
  - выход — 38 параметров (патч) в синтезатор.
- Связи: сплошной поток данных вход → L1 → L2 → L3 → L4 → выход; отдельная входная стрелка от запроса
  к L4 (слова-модификаторы извлекаются из того же запроса параллельно идентичности); из библиотеки
  прототипов — стрелка-обращение в L3 по выбранному номеру прототипа.
- Примечание: на схеме НЕ показываем baseline (сырой retrieval) — он только для сравнения в гл. 4.
