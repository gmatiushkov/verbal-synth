# Дипломный проект: Интеллектуальная система настройки параметров синтеза звука

## Тема
«Разработка интеллектуальной системы для автоматической настройки параметров синтеза звука с применением методов машинного обучения»

---

## Идея и цель

Профессиональные синтезаторы имеют десятки параметров — новичку сложно получить нужный звук без знания теории синтеза. Система принимает текстовый запрос («звонкий колокольчик с длинным эхо») и автоматически настраивает параметры синтезатора.

**Ключевое отличие от диффузионных моделей:** система генерирует не аудиофайл, а набор параметров синтезатора — пользователь может дальше корректировать звук вручную.

---

## Архитектура системы

```
[Текстовый ввод] → [ML-модуль] → [Синтезатор (JUCE/C++)] → [Аудиовыход]
                                          ↑
                            [Физические энкодеры (RPi)]
```

**ML-модуль:** регрессионная модель — text embedding (sentence-transformers / MiniLM) + регрессионная голова. Маппинг: вектор текста → 32 числовых параметра синтезатора.

**Синтезатор:** реализуется на C++ с использованием фреймворка JUCE.

**Деплой:** Raspberry Pi 4 (4GB) — автономное устройство с физическими энкодерами (GPIO), текстовым (в перспективе голосовым) вводом. ML-инференс через ONNX Runtime.

---

## Стек технологий

| Компонент | Технология |
|---|---|
| Синтезатор | JUCE, C++ |
| ML-обучение | Python, PyTorch |
| Text embeddings | sentence-transformers (MiniLM) |
| ML-инференс (деплой) | ONNX Runtime |
| Аудио на RPi | JACK или ALSA |
| Голосовой ввод (опционально) | Whisper.cpp |

---

## Этапы реализации

1. **Этап 1** — Среда разработки: настройка JUCE + IDE, hello-world standalone-приложение, определение формата патча (JSON / struct)
2. **Этап 2** — Синтезатор: реализация всех модулей, отладка, автономная работа без ML
3. **Этап 3** — Датасет: стратегия сбора/синтетической генерации пар «описание → параметры»
4. **Этап 4** — ML-модуль: обучение, валидация, экспорт в ONNX
5. **Этап 5** — Интеграция и UI: полный пайплайн, текстовый ввод, отображение параметров
6. **Этап 6** — Деплой на Raspberry Pi: сборка под ARM Linux, GPIO-энкодеры, опционально STT

## Конфигурация синтезатора

**Тип:** субтрактивный с Ring Modulator и цепочкой эффектов
**Все параметры нормализованы в [0.0, 1.0]**
**Всего параметров: 32**

### Осциллятор 1 (4 параметра)

| # | Параметр | Описание | Диапазон / примечание |
|---|---|---|---|
| 1 | `osc1_morph` | Форма волны | 0=sine, 0.25=triangle, 0.5=saw, 0.75=rsaw, 1=square. Плавный морфинг. |
| 2 | `osc1_octave` | Сдвиг октавы | 0=−2 окт, 0.25=−1, 0.5=0, 0.75=+1, 1=+2. Внутри движка округляется. |
| 3 | `osc1_detune` | Расстройка | 0=−50 центов, 0.5=0, 1=+50 центов |
| 4 | `osc1_level` | Уровень громкости | 0=тишина, 1=максимум |

### Осциллятор 2 (4 параметра)

| # | Параметр | Описание | Диапазон / примечание |
|---|---|---|---|
| 5 | `osc2_morph` | Форма волны (аналогично osc1) | |
| 6 | `osc2_octave` | Сдвиг октавы (аналогично) | |
| 7 | `osc2_detune` | Расстройка | 0=−50 центов, 0.5=0, 1=+50 центов |
| 8 | `osc2_level` | Уровень громкости | |

### Шум и Ring Modulator (2 параметра)

| # | Параметр | Описание | Диапазон / примечание |
|---|---|---|---|
| 9 | `noise_level` | Уровень белого шума | |
| 10 | `ring_mod_amount` | Кольцевая модуляция (osc1 × osc2) | 0=нет, 1=полностью ring-модулированный сигнал. Металлический призвук при расстройке osc2. |

### Фильтр (3 параметра)

| # | Параметр | Описание | Диапазон / примечание |
|---|---|---|---|
| 11 | `filter_type` | Тип фильтра | 0=lowpass, 0.5=bandpass, 1=highpass. Передаётся как непрерывное значение, внутри движка выбирается ближайший режим. |
| 12 | `filter_cutoff` | Частота среза | Логарифмическая шкала: 0→20 Гц, 1→18 кГц |
| 13 | `filter_resonance` | Резонанс / добротность | 0=нет, 1=максимальный |

### Амплитудная огибающая ADSR (4 параметра)

| # | Параметр | Описание | Диапазон / примечание |
|---|---|---|---|
| 14 | `amp_attack` | Время атаки | 0=мгновенно (0.5мс), 1=очень медленно (5с). Логарифмически. |
| 15 | `amp_decay` | Время спада | 1мс..5с |
| 16 | `amp_sustain` | Уровень сустейна | 0=тишина, 1=полная громкость |
| 17 | `amp_release` | Время затухания | 5мс..10с |

### LFO (5 параметров)

| # | Параметр | Описание | Диапазон / примечание |
|---|---|---|---|
| 18 | `lfo_rate` | Частота LFO | 0=очень медленно (0.01 Гц), 1=максимум (20 Гц). Логарифмически. |
| 19 | `lfo_shape` | Форма волны LFO | 0=sine, 0.25=triangle, 0.5=saw, 0.75=rsaw, 1=square |
| 20 | `lfo_to_filter` | Модуляция частоты среза | 0=нет, 1=максимальный размах |
| 21 | `lfo_to_pitch` | Модуляция высоты тона (вибрато) | |
| 22 | `lfo_to_amp` | Модуляция амплитуды (тремоло) | |

### Дисторшн (1 параметр)

| # | Параметр | Описание | Диапазон / примечание |
|---|---|---|---|
| 23 | `drive_amount` | Степень перегруза (soft-clip waveshaper) | 0=чистый сигнал, 1=сильный перегруз |

### Phaser (3 параметра)

| # | Параметр | Описание | Диапазон / примечание |
|---|---|---|---|
| 24 | `phaser_rate` | Скорость фейзера | 0.01..8 Гц |
| 25 | `phaser_depth` | Глубина фейзера | |
| 26 | `phaser_feedback` | Обратная связь фейзера | |

### Delay (3 параметра)

| # | Параметр | Описание | Диапазон / примечание |
|---|---|---|---|
| 27 | `delay_time` | Время задержки | 0=10мс, 1=1с |
| 28 | `delay_feedback` | Обратная связь | 0=один повтор, 1=бесконечные повторы |
| 29 | `delay_mix` | Уровень дилея | 0=сухой сигнал, 1=только обработанный |

### Reverb (3 параметра)

| # | Параметр | Описание | Диапазон / примечание |
|---|---|---|---|
| 30 | `reverb_time` | Время реверберации | 0=очень короткое, 1=максимальное |
| 31 | `reverb_damp` | Демпфирование (яркость хвоста) | 0=яркий хвост, 1=тёмный/приглушённый |
| 32 | `reverb_mix` | Уровень реверберации | 0=сухо, 1=полностью смоченный |

## Порядок модулей в цепочке сигнала

```
OSC1 ──┐
OSC2 ──┼──► Ring Mod ──► Mix ──► Filter ──► Amp (ADSR) ──► Drive ──► Phaser ──► Delay ──► Reverb ──► Out
Noise ─┘
              ↑              ↑         ↑
             LFO           LFO       LFO
          (to_pitch)   (to_filter) (to_amp)
```

---

## Параметры вне ML-вектора (управляются MIDI / UI)

- `midi_note` — высота ноты (MIDI 0..127)
- `midi_velocity` — динамика (MIDI 0..127)
- `midi_duration` — длительность ноты

## Примеры маппинга (для тестирования / датасета)

**«Металлический колокол»**
```
osc1_morph=0, osc2_morph=0, osc2_detune=0.7, ring_mod_amount=0.85,
amp_attack=0, amp_decay=0.55, amp_sustain=0, amp_release=0.7,
reverb_time=0.5, reverb_mix=0.4
```

**«Мягкий пэд»**
```

ring_mod_amount=0, filter_cutoff=0.35, filter_resonance=0.1,
amp_attack=0.6, amp_sustain=0.85, amp_release=0.75,
reverb_time=0.6, reverb_mix=0.35
```

**«Жёсткий синт-бас»**
```
osc1_morph=0.5, osc2_morph=0.5, filter_type=0, filter_cutoff=0.3,
filter_resonance=0.6, amp_attack=0, amp_decay=0.3, amp_sustain=0.5,
drive_amount=0.55
```

**«Вибрирующий лид»**
```
osc1_morph=0.5, filter_cutoff=0.6, filter_resonance=0.4,
amp_attack=0.1, amp_sustain=0.8, amp_release=0.2,
lfo_rate=0.35, lfo_shape=0, lfo_to_pitch=0.15
```
