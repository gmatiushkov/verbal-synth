Дизайн v4 (A6-Claude) — с фокусом на понимание языка
Сначала — как модель вообще «понимает слово» (чтобы дизайн бил в цель)
Модель = замороженный энкодер (слово → вектор смысла; синонимы рядом: «резкий»≈«острый»≈«sharp») + обучаемая голова (вектор → параметры). Чтобы голова связала «резкий» → быструю атаку, в данных должны быть примеры, где это слово стоит рядом с патчами с быстрой атакой. Отсюда три следствия, на которых строится весь подход:

Энкодер даёт бесплатное обобщение ВНУТРИ смыслового кластера. Покрыли «резкий» — «острый/колкий» подтянутся сами. Значит надо покрыть не каждое слово, а каждый отдельный смысловой кластер, привязанный к характеристике.
Связь «слово → ручка» рождается только из данных. Энкодер не знает, что «резкий» относится к атаке. Поэтому каждый дескриптор-кластер должен много раз встретиться привязанным к своей области параметров.
Связь many-to-many, и это нормально. «Резкий» — это и быстрая атака, И яркость, И агрессия. Навязывать 1:1 нельзя. Модель разрулит по контексту и по статистике совстречаемости. Наша задача — дать правильную статистику, а не словарь-однозначник.
→ Вывод: главный артефакт v4 — не патчи, а богатое, повторяющееся покрытие пространства дескрипторов, привязанное к параметрам.

Три артефакта v4
1. Схема атрибутов + таблица заземления (скелет)
~11 перцептивно-функциональных осей, у каждой 3-5 уровней, и каждый уровень привязан к конкретной области параметров:

Ось	Уровни	Куда в параметрах
Роль/идентичность	бас, суб, лид, пэд, плак, кейс, орган, колокол, духовой, струнный, хор, дрон, kick/snare/hat, fx…	банк + база октавы + архетип огибающей + конфиг осц
Регистр	суб/низ/сред/высокий/оч.высокий	osc1_octave
Яркость	тёмный/тёплый/нейтр/яркий/пронзительный	lp_cutoff (+drive_tone)
Атака	мгновенная/быстрая/средняя/мягкая/наплыв	amp_attack
Тело-длина	стаккато/щипок/сустейн/длинный/дрон	amp D/S/R
Фильтр-движение	статичный/лёгкий свип/вау-щипок/сильный свип	fenv_amount+времена, резонанс
Движение (LFO)	статика/вибрато/тремоло/вобл/морф/Лесли	LFO маршрут+форма(синус=натурально)+скорость
Текстура/драйв	чисто/насыщ./грязно/дисторшн/шумно	drive_amount, mix_noise
Пространство	сухо/комната/зал/бескрайне	reverb
Толщина	тонко/унисон-толсто/октавный стек	mix_osc2, detune, semitones(±12 только для стека)
Характер	тёплый, холодный, тревожный, эпичный, мечтательный, агрессивный…	мягкие сдвиги нескольких
«Звук» = роль-якорь + значения осей. Роль задаёт идентичность (банк/октава/огибающая), оси её модулируют в разумных границах. Всё, что спека не трогает → нейтраль (A3: semitones=0, LFO/детюн/HP выкл).

2. Роль-шаблоны
Каждая роль → базовые параметры + какие оси ей разрешено варьировать и в каких пределах (бас не уходит в высокий регистр; колокол не бывает в суб-басу). Это вшивает доменное знание и убирает бессмысленные комбинации.

3. ⭐ Дескрипторный лексикон (ядро понимания языка)
Для каждого уровня каждой оси — богатый набор слов/фраз во ВСЕХ регистрах языка:

Прямые/технические: «быстрая атака», «пилообразная волна», «треугольный LFO на питч»
Психоакустические/метафоры: быстрая атака → «резкий, колкий, цепкий, щёлкающий, ударный, мгновенный»; мягкая → «наплывающий, вырастает, дышит, нежно вступает»
Кросс-сенсорные: «тёплый, холодный, воздушный, густой, прозрачный, острый, мягкий, стеклянный, бархатный, жидкий, свистящий»
Ассоциативные/инструменты: «как орган», «виолончельный», «колокольный», «ламповый»
Настроение/роль: «тревожный, мечтательный, агрессивный», «бас для трэпа», «лид для транса»
Лексикон many-to-many: «воздушный» живёт под яркостью+шумом+пространством; «свистящий» под ролью-флейта+узкий+высокий. Черновик генерит Claude (он силён в языке), мы курируем. Это разовый артефакт.

Как описания обеспечивают полное покрытие (механизм A2+A5)
Для каждой спеки Claude пишет K разных описаний, обязанных охватить спектр: хотя бы одно техническое, одно метафорическое, одно ассоциативно-инструментальное, одно настроенческое — черпая слова из лексикона по уровням спеки + добавляя свободные перефразы. Плюс критично для реального юзера:

Разная длина и полнота: от 2 слов («тёплый бас», «орган») до развёрнутой фразы. Реальные люди пишут коротко — данные должны это отражать.
Только роль/инструмент («колокольчик»), только настроение («тревожный звук»), смешанные, англо-вкрапления, разговорно, с опечатками.
Трекер покрытия: после генерации проверяем, какие дескрипторы/кластеры недопредставлены, и догенерируем точечно — чтобы каждый кластер встретился достаточно раз (бюджет покрытия), а не «как повезёт».
Так каждое слово из лексикона много раз привязано к своей области параметров → модель учит ВЕСЬ спектр, а не только «быструю атаку».

Поток генерации v4
attributes.json (оси+уровни+заземление) → 2. roles.json (шаблоны) → 3. lexicon.json (дескрипторы, Claude+курация) → 4. сэмплер валидных спек (роль×оси, с ограничениями) со сбалансированным покрытием → 5. на спеку Claude рендерит параметры (идентичность роли + заземление + нейтраль) и K описаний (спектр+разная длина) → 6. param_convert→норма, QA-консистентность идентичности, снап нейтрали → 7. dataset.jsonl → обучение (+B1 маскирование неактивных, +B2 semitones-классификацией).
